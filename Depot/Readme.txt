*** 1.Описание сущностей. ***
Сущности понимаемые системой делятся на три класса:
1. Знания.
2. Данные.
3. Служебные сущности.

** Знания. **
Это правила по которым система интерпретирует, создает и преобразует данные и знания.

** Данные. **
Это информация, которой оперирует система.

** Служебные сущности. **
К служебным сущностям относятся домены и права пользователей.

Все три класса сущностей делятся на два типа:
1. Факты.
2. Правила.

*** 2.Язык системы. ***
1. Факты.
2. Правила.
3. Данные.
4. Выборка информации.

** Факты **
Пример знания-факта о enum.
SET LexemEnum AS ENUM (lxmSET, lxmAS);
Пример знания-факта о константе
SET DataBaseName AS CONST STRING;

* Представление о фактах знаний в виде супер-таблиц *
1. Домены, метатип - 1, тип элемента (StructureNum) - 1.
2. Константы, метатип - 1, тип элемента - 2.
3. Перечисления, метатип - 1, тип элемента - 3.
4. Таблицы, метатип - 1, тип элемента - 4.
5. Учеты, метатип - 1, тип элемента - 5.

*** 2. Описание базы данных. ****
База данных физически хранит на диске страницы содержащие trait-ы - атомарные кусочки информации.

** Trait **
Trait бывает прямой и индексный и имеет следующую структуру:
Прямой:
1. Мета тип элемента - байт < 128
2. Тип элемента к которому принадлежит Trait - ushort (итого 65536 типов)
3. Ключ элемента
4. мета тип поля - байт
5. Код поля в элементе - ushort (итого 65536 полей в лдном элементе)
6. Данные

Обратный:
1. Мета тип элемента - байт >= 128
2. Тип элемента к которому принадлежит Trait - ushort (итого 65536 типов)
3. мета тип поля - байт
4. Код поля в элементе - ushort (итого 65536 полей в лдном элементе)
5. Данные
6. Ключ элемента

** Ключ элемента **
Ключ элемента имеет размер 8 байт и тип long и структурно состоит из:
 1. номер элемента
 2. номер строки в элементе (опционально)
 3. номер транзакции внутри элемента (опционально)
